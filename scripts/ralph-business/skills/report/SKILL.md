# Report Skill

ビジネスレポート・ドキュメント作成タスクを実行する。

---

## The Job

1. 目的・対象読者を確認
2. 必要な情報・データを収集
3. 構造化されたドキュメントを作成
4. 指定フォーマットで保存

---

## レポートタイプ

### 1. 調査レポート
研究・調査結果をまとめる

```markdown
# {タイトル}

## エグゼクティブサマリー
## 背景・目的
## 調査手法
## 調査結果
## 結論・提言
## 付録
```

### 2. 提案書・企画書
新規施策・プロジェクトを提案

```markdown
# {提案タイトル}

## 提案概要
## 背景・課題
## 提案内容
## 期待効果
## 実施計画
## 必要リソース
## リスクと対策
```

### 3. 経営サマリー
意思決定者向けの簡潔なまとめ

```markdown
# {タイトル}

## エグゼクティブサマリー
{300字以内}

## 主要な発見
- ポイント1
- ポイント2
- ポイント3

## 推奨アクション
1. {最優先アクション}
2. {次に重要なアクション}
3. ...

## 次のステップ
```

### 4. 進捗レポート
定期的な状況報告

```markdown
# {期間} 進捗レポート

## サマリー
## 完了事項
## 進行中の事項
## 課題・リスク
## 次期の計画
## KPI状況
```

## ライティングガイドライン

### 読みやすさ

- **結論ファースト**: 最初に結論、その後に根拠
- **箇条書き活用**: 長い文章より箇条書き
- **1段落1メッセージ**: 1つの段落に1つの主張
- **具体的な数字**: 「多い」ではなく「30%増」

### 構造

- **見出し階層**: H1 > H2 > H3 の階層を守る
- **目次**: 長いドキュメントには目次を追加
- **視覚的要素**: 表、図、箇条書きを活用

### トーン

- **ビジネスライク**: 丁寧だが簡潔
- **客観的**: 意見より事実・データ
- **アクション志向**: 「〜すべき」「〜を推奨」

## フォーマット別の出力

### Markdown (.md)
デフォルト。最も汎用的。

### Obsidian
フロントマター付きで保存:

```markdown
---
createdAt: YYYY-MM-DD HH:MM
tags: 
- ralph_business
- report
---

# {タイトル}
...
```

### Notion
Notion MCP経由で保存。
ページ構造はMarkdownと同様。

## acceptanceCriteria対応

| Criteria | 確認方法 |
|----------|----------|
| 「エグゼクティブサマリー」 | セクションが存在し、指定文字数以内 |
| 「目次を含む」 | 目次セクションが存在 |
| 「N項目以上」 | 該当セクションにN個以上の項目 |
| 「表形式」 | Markdown表が含まれている |
| 「指定ファイルに保存」 | ファイルが存在する |
